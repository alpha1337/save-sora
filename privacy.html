Save Sora

Privacy Policy

Save Sora is a Chrome extension that helps users review and selectively download their own Sora published videos and drafts from their logged-in Sora account for backup.

Last updated: March 30, 2026

What It Accesses How It Uses Data Local Storage What It Does Not Collect Sharing Third-Party Content Your Choices Contact
Short version: Save Sora processes your Sora video data locally in your browser so you can review and download your own files.
No developer backend: the current build does not send your Sora library, auth tokens, or cookies to a developer-owned server.
Local state only: extension storage is used only for settings and local UI state such as renamed titles, selection state, and downloaded state.

What Save Sora Accesses

To perform its single purpose, Save Sora accesses the following information from the user's existing logged-in Sora session:

  • Sora video titles, prompts, thumbnails, metadata, and downloadable video references.
  • The user's own published videos and drafts available through the current Sora browser session.
  • OpenAI-hosted video and thumbnail URLs needed to preview or download items the user selects.

How Save Sora Uses Data

Save Sora uses that information only to support the extension's single purpose.

  • Build a review list of the user's own Sora published videos and drafts.
  • Let the user search, sort, rename, and selectively choose files for backup.
  • Download only the videos the user explicitly selects.

Save Sora uses the user's existing browser session to access Sora. It does not require the user to paste credentials or manually provide tokens.

Local Storage And State

Save Sora uses chrome.storage.local only for local extension settings and local UI state.

  • Theme preference.
  • Default source and sort order.
  • Maximum videos setting.
  • Renamed titles.
  • Selection state.
  • Removed state and downloaded state.
  • Current fetch and download queue state so the popup can recover after being closed.

What Save Sora Does Not Collect

  • Passwords.
  • Personal account credentials.
  • Auth tokens collected by the developer.
  • Cookies collected by the developer.
  • Analytics or telemetry sent to the developer in the current build.
  • Your Sora library uploaded to a developer-owned backend.

Important distinction: the extension uses the auth context already present in the user's own Sora tab to read the user's own data locally. That is not the same as the developer collecting or receiving those credentials.

How Data Is Shared

  • Save Sora does not sell user data.
  • Save Sora does not transfer user data to third parties for advertising.
  • Save Sora does not use or transfer user data for purposes unrelated to the extension's single purpose.
  • Save Sora does not use or transfer user data to determine creditworthiness or for lending purposes.

Third-Party Content And Services

  • Sora data is accessed from OpenAI-operated domains needed for the extension's single purpose.
  • Selected video downloads come from OpenAI-hosted Sora video URLs.
  • The Donate tab links to and embeds Ko-fi content if the user chooses to view it.

Ko-fi content is separate from the extension's core logic for listing and downloading the user's Sora files.

Your Choices And Control

  • You can remove the extension at any time.
  • You can clear extension data through Chrome's extension settings.
  • You can choose which videos to download instead of downloading everything automatically.
